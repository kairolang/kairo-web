# `libcxx`

The `libcxx` namespace encompasses the C and C++ standard library headers for use in the current version of Kairo.

## C Standard Library Headers

### Cross-Platform C Headers
  - [`<fcntl.h>`](https://en.cppreference.com/w/c/io) - File control options
  - [`<malloc.h>`](https://en.cppreference.com/w/c/memory/malloc) - Memory allocation (non-standard)
  - [`<memory.h>`](https://en.cppreference.com/w/c/string/byte) - Memory operations (non-standard)
  - [`<sys/stat.h>`](https://en.cppreference.com/w/c/io) - File status
  - [`<sys/timeb.h>`](https://en.cppreference.com/w/c/chrono) - Time utilities
  - [`<sys/types.h>`](https://en.cppreference.com/w/c/types) - System types
  - [`<sys/utime.h>`](https://en.cppreference.com/w/c/chrono) - File time modification

### Windows-Specific C Headers

  - `<corecrt_search.h>` - Core runtime search (Microsoft-specific)
  - `<corecrt_share.h>` - Core runtime sharing (Microsoft-specific)
  - `<corecrt_startup.h>` - Core runtime startup (Microsoft-specific)
  - `<corecrt_stdio_config.h>` - Core runtime stdio config (Microsoft-specific)
  - `<corecrt_terminate.h>` - Core runtime termination (Microsoft-specific)
  - `<corecrt_wconio.h>` - Core runtime wide console I/O (Microsoft-specific)
  - `<corecrt_wctype.h>` - Core runtime wide character types (Microsoft-specific)
  - `<corecrt_wdirect.h>` - Core runtime wide directory (Microsoft-specific)
  - `<corecrt_wio.h>` - Core runtime wide I/O (Microsoft-specific)
  - `<corecrt_wprocess.h>` - Core runtime wide process (Microsoft-specific)

  - `<direct.h>` - Directory control (Microsoft-specific)
  - `<dos.h>` - DOS interface (Microsoft-specific)
  - `<fpieee.h>` - IEEE floating point (Microsoft-specific)
  - `<io.h>` - Low-level I/O (Microsoft-specific)

  - `<mbstring.h>` - Multibyte string (Microsoft-specific)

  - `<share.h>` - File sharing (Microsoft-specific)
